# Roadmap

Scenario Form is currently in MVP stage.

This roadmap shows the current direction of the project. Priorities may change based on real user feedback and issues found on complex web forms.

## Related Links

- [Privacy Policy](./PRIVACY.md)
- [Support](./SUPPORT.md)

## v0.1.0 — MVP

- [x] Capture form values from a web page
- [x] Replay saved form scenarios
- [x] Manage saved scenarios
- [x] Configure flexible extension settings
- [x] Configure custom field detection rules
- [x] Use built-in presets for complex form platforms
- [x] Support nested and custom form controls
- [x] Highlight detected and filled form fields
- [x] Export and import scenarios and rules
- [x] Store scenarios and rules locally in the browser
- [x] Add public bug report templates
- [x] Provide public [Privacy Policy](./PRIVACY.md) and [Support](./SUPPORT.md) documentation

## Current Focus

- [ ] Stabilize the first public release
- [ ] Improve first-use experience
- [ ] Collect feedback from early users
- [ ] Fix issues found on real forms
- [ ] Improve reliability on dynamic web applications

## Planned Improvements

- [ ] Improve field detection and matching
- [ ] Improve rule testing and debugging experience
- [ ] Improve scenario management UX
- [ ] Improve presets for popular form platforms
- [ ] Improve support for multi-step forms
- [ ] Improve support for complex custom controls

## Under Consideration

These ideas are being considered, but are not guaranteed for a specific release.

- [ ] Advanced date picker support
- [ ] Optional diagnostics export for bug reports
- [ ] More flexible import/export options
- [ ] Better onboarding for non-technical users
- [ ] More preset coverage for popular form platforms

## Privacy Direction

Scenario Form is designed to keep user data local by default.

The project does not plan to send form values, scenario values, page content, passwords, cookies, or personal data to external servers.
